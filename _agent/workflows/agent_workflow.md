# 📋 智能助手与 Bot 程序交互操作手册

> 本文档描述 AI 智能助手在维护和更新 Bot 程序时应遵循的操作标准、业务守则及核心流水线机制。

---

## 🔴 黄金守则：操作安全红线

> [!IMPORTANT]
> **严禁在 Bot 运行状态下修改核心代码 (`src/`)！**
> **所有交互界面、文档说明及与用户的交流必须使用全中文。**

**标准停机流程：**

1. 执行 `python src/utils/send_offline.py` → 向管理员发布下线通报。
2. **锁检查**：确认 `data/sessions/` 目录下的 `.lock` 文档已释放。

**改动后的标准上线流程：**

1. 运行 `start_bot.bat` 重新上线。
2. **RunID 核对**：检查管理员收到的私信，对比新的 `RunID`。
3. **冒烟测试**：转发一条资源测试模式四 UI 连通性。

---

## 🎨 工作模式三：数据分拣流水线 (The Discovery Pipeline)

这是本项目的核心业务链。它并非全自动，而是一套**基于双重校验的人机协作增量处理机制**。

### 1️⃣ 步骤一：新鲜度感知与 Skip 逻辑

当用户触发 `[🔄 更新词库]` 时，Bot 会自动进行新鲜度对账：

- **逻辑 ID 校验**：比对 `backup_id` (如 B3, B4)。
- **物理时间戳校验 (V2.2)**：比对备份文件夹的 `Max MTime`。
- **结论**：
  - **Match**：跳过扫描，直接提供 "🚀 恢复界面" 按钮。
  - **Mismatch**（检测到新备份）：自动启动 P0/P1 重跑，确保候选池永不过时。

### 2️⃣ 步骤二：候选词挖掘 (P1 - Discovery)

- **核心程序**：`entity_extractor.py`。
- **操作**：程序会扫描全量备份 JSON，自动剔除 `entities.json` 中已有的“熟词”，挖掘高频未实体的潜在 Creator/Actor/Tag。
- **产出**：发送包含 Web 管理链接的消息。同步生成 `docs/entities/[bot]/candidates/` 分卷作为备考。

### 3️⃣ 步骤三：交互分拣与结算 (P1.5 - Web Sorter)

这是人机回环的关键：

1. **勾选分类**：勾选 C (Creator), A (Actor), T (Tag), N (Noise)。
2. **提交决策**：
   - **💾 暂存 (Staging)**：仅存入本地暂存 JSON，UI 显示蓝色虚线。不触发全量打标。
   - **🚀 确认成交**：自动触发：**同步规则 (Sync) → 清理物理池 (Cleanup) → 全量打标 (P2) → 预览导出 (P3)**。
3. **闭环刷新**：提交后网页自动刷新，已完成词条物理消失，序号重新排齐。

## 🛠️ 工作模式四：人工补全流程 (Manual Mode 4)

- **触发方式**：转发私密库消息给 Bot。
- **四字段管理**：
  - 1️⃣ **创作者**：支持实时追加或合并别名。
  - 2️⃣ **女m**：实时更新人物映射。
  - 3️⃣ **关键词**：增加特征打标。
  - 4️⃣ **补充描述**：补充自动化无法识别的文案信息。
- **精准识别**：支持通过转发同步页眉文本特征实现 100% 自动关联数据库记录。

---

## 📁 机制变更守则

当系统**路径、文件结构或 API 行为**发生变化时，必须同步更新：

- `bot_system.md`：更新目录树、数据流。
- `agent_workflow.md`：（本文档）更新操作步骤、Skip 逻辑。
- `tg_mechanics.md`：更新 Telegram 底层 API 或资源链解释。

> [!CAUTION]
> **多 Bot 隔离警示**：所有操作必须明确当前 Bot 环境，确保对应的是 `docs/entities/[target_bot]/` 的数据。
